Why...

did I create it?

Designed to scratch my own itch while developing a non dynamic site. I wanted SASS and CoffeeScript. I wanted HAML for my content. Eventually running sass, coffeescript and a build script became enough of a hindrance that I started to think bigger, and Suite was born.

would you use it?

Automatically compile coffeescript, sass etc.

Separate files logically (contact.js, core.js, menu.js)

Compiles for deploy, minify and compress

Compiles to fingerprinted files (screw you caches)

Pretty much it’s the Rails asset pipeline

This sounds awesome

How do I use it?

gem install suite
(yes, it’s in ruby)

suite project name

Creates a new project in the folder name. Prompts for some config, creates assets, config and content directories.

suite server

Run in a project folder. Starts a development server at 0.0.0.0:9000. Pages and assets are rendered on the fly.

suite build

Run in a project folder. Outputs to build folder. Compiles pages to static html. Compiles assets to fingerprinted files (e.g. 79c61759396ec4e3498452b7d3675dff.js).

JS and CSS are minified and compressed according to your config.yml.
